## Column-Level Source Map

Every column in `feature_tape_v2` is documented below with its source type, null semantics, and analysis scope.

**Source type key:**

| Source Type | Meaning |
|------------|---------|
| `snapshot_native` | Read directly from `universe_snapshot` |
| `micro_native` | Read directly from `microstructure_log` |
| `derived_at_collection` | Computed in `feature_tape_v2.py` at collection time from one or more source columns |
| `fire_level_aggregate` | Computed once per fire from all rows; identical across all rows in a fire |
| `system` | Generated by the collection script (IDs, timestamps, source flags) |

**Analysis scope key:**

| Scope | Meaning |
|-------|---------|
| `full_universe` | Available for all rows (eligible + ineligible) |
| `eligible_only` | Available for eligible rows only; expected NULL for ineligible rows |
| `micro_only` | Available only when a matching `microstructure_log` row exists; NULL otherwise |
| `both` | Available for all rows AND meaningful for both eligible and ineligible analysis |

### Classification Columns

| Column | Type | Source Type | Null Semantics | Scope |
|--------|------|------------|----------------|-------|
| `lane` | TEXT | derived_at_collection | Never NULL; derived from `eligible` + `gate_reason` + `pool_type` | both |
| `lane_source` | TEXT | system | Never NULL; always `'derived_at_collection'` | both |
| `venue` | TEXT | derived_at_collection | NULL if pool_type is unrecognised | both |
| `pool_type` | TEXT | snapshot_native | Rarely NULL; e.g., `'pumpswap_cpmm'`, `'raydium_amm_v4'` | both |
| `pumpfun_origin` | INTEGER | snapshot_native | NULL if not applicable; 1 if token originated on PumpFun | both |
| `eligible` | INTEGER | snapshot_native | Never NULL for collected rows; 1 = passed all scanner gates, 0 = failed | both |
| `gate_reason` | TEXT | snapshot_native | NULL if `eligible = 1`; scanner-provided reason string if `eligible = 0` | both |

**`lane` / `universe_category` semantics:**

The `lane` column is a **universe category** (venue-state classification), not a strategy lane. For eligible rows (`eligible = 1`), it encodes the venue: `pumpswap_live`, `raydium_live`, `orca_live`, `meteora_live`, or `other_live`. For ineligible rows (`eligible = 0`), it encodes the rejection class: `spam_filtered`, `impact_filtered`, `age_filtered`, `vol_filtered`, `liq_filtered`, or `ineligible` (catch-all).

**`eligible` and `gate_reason` semantics:**

These are the authoritative columns for filtering between primary (eligible-only) and secondary (full-universe) analysis scopes. The `eligible` column is the single boolean gate. The `gate_reason` column provides the human-readable reason for rejection when `eligible = 0`. When `eligible = 1`, `gate_reason` is NULL.

**Model training vs audit:**

| `eligible` | Intended Use |
|-----------|-------------|
| 1 | Model training, feature sweeps, promotion gate evaluation, holdout evaluation |
| 0 | Audit only: coverage monitoring, lane distribution, data quality checks, missingness analysis |

Ineligible rows must **never** be included in model training targets or feature sweep primary results unless explicitly documented and justified.
